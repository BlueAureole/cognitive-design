认知设计框架包  

# 简介
程序设计的目的是实现功能。功能越复杂，程序包含的逻辑也越复杂。认知设计的观点是，语言直接描述对复杂业务的认知，而把功能的实现交给系统框架。

# 框架特性
## 性能表现
- 过程式的耗时：每次数据交互时长的累加。
- 认知式的耗时：每个依赖交互批次的最大值累加。
- 附图：![性能表现对比图](https://image.zhiwei.xyz/paragraph/img/660684363226295296_1_8_G3I1L.jpg)

## 代码可读性
- 过程式编程：不透明依赖
- 认知式编程：透明依赖
